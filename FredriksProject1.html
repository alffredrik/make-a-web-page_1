Lesson 1: The Basics of the Web and HTML

The Major Pieces
At the start of this class we discussed how end users, computers, servers, http, and the internet all fit together. The key takeaway was the concept of the internet, the world largest computer network, and how we through the internet, starting with a request through a browser on our computer, using HTTP, can access information stored on servers all over the world.
The WWW
The World Wide Web is a collection of HTML documents. HTML, HyperText Markup Language is the foundation to build everything we access on the WWW. We can find any kind of file (media, text etc.) on the web but HTML is the is building block of the actual webpages. Other file types or links are embedded into the HTML document (or webpage).
HTML
HTML is made up of text content, markup, references to other docs and links to other pages. Within the HTML document or webpage we have a series of HTML Elements which are components of the document that are usually started and ended with a tag. Examples of tags are <b> for bold lettering or <em> for emphasized letters. It is important to always add an end tag i.e. </b> or everything that follows the opening tag will be affected. There are some elements that does not require a closing (or end) tag and these are called void elements. The break <br> tag is an example of a void element. Another important concept is block vs. inline elements. Block elements start and end with a new line, as with <p> and <div>. Block elements form a "box" aroundthe content of the element. Inline elements are displayed without line breaks. Examples of Inline elements are <b>, <em> and <img>. Lastly we noted that all whitespace, new lines, tabs and spaces all turn into a single space (regardless of how lengthy) within the HTML document.To force line changes we can use the break <br> or paragraph <p> tags.
Final Thoughts
Computers are stupid, people are smart. But people (programmers) make mistakes and computers do not. Thus the onus is on the us, the programmers, to make sure we tell the computers exactly what we need them to do, down to the last detail. Any errors in the input means the output will be corrupted.

Lesson 2: Creating a Structured Document with HTML

The Major Pieces part II
HTML (HyperText Markup Language), CSS (Cascading Style Sheets) and DOM (Document Object Model) are the focus of this lesson. Javascript was mentioned but will not be covered. HTML is the markup language used to build the webpage i.e. the structure. CSS is used for adding color, format and style to the HTML document. The Document Object Model (DOM) provides a structured representation of the HTML document. It gives us a representation of the document as a structured group of nodes and objects. The nodes of every document are organized in a tree structure, called the DOM tree. We used the Chrome Developer Tools option to look at the structure of a web page, showing it as a DOM tree.
Javascript is used to add interactivity to a webpage.
Boxes, boxes and...wait for it...boxes!
Everything on a webpage is boxes. We did an exercise to identify the number of separate boxes on a web page and we learned that we can divide the page in to more or less boxes, depending on how much dept we want to go to. Basically a box represents an element and depending on how granular we want to be able to control the contents on our webpage we can break it down in more or less elements/boxes.
Editors
Sublime is a powerful text editor for programming and it is recommended for more advanced programming. Codepen seems to fit our needs best at this stage, due to it's ease of use and direct feedback.

Lesson 3: Adding CSS Style to HTML Structure

Repetition = bad
The fewer places you have to touch to make a change to multiple parts of your webpage the better it is. Less repetition mean easier to manage code and less chance of you missing something when you make changes to your webpage i.e. fewer places to update. It should also help with structure of the code and make it more readable. By using CSS you can define style and color for different components of your webpage and every time you use a certain component it will get it's style from the CSS file. No need to define the style again each time.
CSS
Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language. Cascading means Most Specific Rule applies. Keep going down and down in the rules until you find the one that best describes the elements. That rule gets applied. If we don't have a CSS file the Web browser will display our page according to a default style template associated with that particular browser.
